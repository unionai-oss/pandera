From pandera/api/function_dispatch.py:
```python
class Dispatcher:
    _function_registry: dict[type, Callable]
    def register(self, fn): ...   # keys on first-arg type annotation
    def __call__(self, *args, **kwargs): ...  # dispatches on type(args[0])
```

After Phase 5, builtins are keyed on nw.Expr — patch target changes from
`_function_registry[nw.LazyFrame]` to `_function_registry[nw.Expr]`.

New builtin calling convention (post-Phase 5):
```python
# For column check (key is a real column name):
expr = check_fn(nw.col(key))   # check_fn receives nw.Expr, returns nw.Expr

# For frame check (key == "*"):
expr = check_fn(frame)          # check_fn receives nw.LazyFrame/nw.DataFrame, returns nw.Expr
```

New native=False user check convention (post-Phase 5):
```python
# Currently: user_check(frame, key) — returns nw.LazyFrame
# After:     user_check(nw.col(key)) — returns nw.Expr   [for column checks]
#            user_check(frame)       — returns nw.Expr   [for key=="*"]
```
