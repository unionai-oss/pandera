Structure of the file:

```python
"""Container-level tests for the narwhals backend.

Tests cover:
  CONTAINER-01: NarwhalsSchemaBackend has failure_cases_metadata, drop_invalid_rows
  CONTAINER-02: schema.validate(pl.DataFrame/pl.LazyFrame) works end-to-end
  CONTAINER-03: strict=True raises SchemaError; strict="filter" drops extra columns
  CONTAINER-04: lazy=True collects all errors before raising SchemaErrors
  REGISTER-01: register_narwhals_backends() is idempotent via lru_cache
  REGISTER-02: DataFrameSchema backend for pl.DataFrame/pl.LazyFrame is narwhals after registration
  REGISTER-04: narwhals backend is NOT registered by default
  TEST-03: SchemaError.failure_cases is a native pl.DataFrame, not nw.DataFrame
"""

import pytest
import polars as pl
import narwhals.stable.v1 as nw
from pandera.api.polars.container import DataFrameSchema
from pandera.api.polars.components import Column
from pandera.api.checks import Check
from pandera.errors import SchemaError, SchemaErrors
```

Stubs to create (all decorated `@pytest.mark.xfail(reason="...", strict=False)`):

**test_failure_cases_metadata (CONTAINER-01)**
- Import NarwhalsSchemaBackend from pandera.backends.narwhals.base inside the test body
- Create a minimal SchemaError with a pl.DataFrame as failure_cases
- Call `NarwhalsSchemaBackend().failure_cases_metadata("test", [err])`
- Assert the return value has `.failure_cases` attribute (is a pl.DataFrame)

**test_drop_invalid_rows (CONTAINER-01)**
- Import NarwhalsSchemaBackend from pandera.backends.narwhals.base inside the test body
- Create a simple nw.LazyFrame and a mock ErrorHandler with one collected error
- Verify drop_invalid_rows returns a frame (does not raise)

**test_validate_polars_dataframe (CONTAINER-02)**
- Register narwhals backends inside test body using pandera.config.config_context
- Create schema with one Column(pl.Int64, name="a")
- Call schema.validate(pl.DataFrame({"a": [1, 2, 3]}))
- Assert result is pl.DataFrame (native, not narwhals wrapper)

**test_validate_polars_lazyframe (CONTAINER-02)**
- Same as above but with pl.LazyFrame({"a": [1, 2, 3]})
- Assert result is pl.LazyFrame

**test_validate_invalid_raises_schema_error (CONTAINER-02)**
- Schema with Column(pl.Int64, name="a", checks=[Check.greater_than(10)])
- Call schema.validate(pl.DataFrame({"a": [1, 2, 3]})) inside try/except SchemaError
- Assert SchemaError is raised

**test_strict_true_rejects_extra_columns (CONTAINER-03)**
- Schema with strict=True, columns={"a": Column(pl.Int64)}
- Validate pl.DataFrame({"a": [1], "b": [2]}) — "b" is extra
- Assert SchemaError or SchemaErrors raised

**test_strict_filter_drops_extra_columns (CONTAINER-03)**
- Schema with strict="filter", columns={"a": Column(pl.Int64)}
- Validate pl.DataFrame({"a": [1], "b": [2]})
- Assert result has only column "a"

**test_lazy_mode_collects_all_errors (CONTAINER-04)**
- Schema with two columns with failing checks
- Validate with lazy=True
- Assert SchemaErrors raised containing multiple errors

**test_register_is_idempotent (REGISTER-01)**
- Import register_narwhals_backends from pandera.backends.narwhals.register inside test body
- Call register_narwhals_backends() twice
- Assert no exception and registry state is consistent (second call is lru_cache no-op)

**test_polars_backends_registered (REGISTER-02)**
- After calling register_narwhals_backends(), verify that DataFrameSchema's backend for pl.DataFrame is DataFrameSchemaBackend (not PolarsSchemaBackend)
- Import both backends inside the test body for isinstance check

**test_narwhals_not_registered_by_default (REGISTER-04)**
- Without calling register_narwhals_backends(), check BACKEND_REGISTRY key (DataFrameSchema, pl.DataFrame)
- Import PolarsSchemaBackend and assert registered backend is NOT NarwhalsSchemaBackend
- Use a fresh import — do not rely on module-level state

**test_failure_cases_is_native (TEST-03)**
- With narwhals backend registered, validate an invalid pl.DataFrame
- Catch SchemaError
- Assert isinstance(err.failure_cases, pl.DataFrame) is True OR err.failure_cases is not an instance of nw.DataFrame

All stubs use `strict=False` xfail so they collect and pass without implementation yet.
