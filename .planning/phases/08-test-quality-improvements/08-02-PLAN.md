<objective>
Implement TQ-02: fix the silent `pl_items` drop in the polars branch of `_concat_failure_cases` in `pandera/backends/narwhals/base.py`, and add a regression test that asserts the merged behavior.

Purpose: A polars narwhals validation run with mixed failure types (schema-level errors via `_build_eager_failure_case`/`_build_scalar_failure_case` producing `pl.DataFrame` items, plus data-check failures on lazy polars frames via `_build_lazy_failure_case` producing `nw.LazyFrame` items) currently loses the schema-level failure rows. The polars branch only concatenates `nw_items` and discards `pl_items` without warning. Fix: collect the lazy narwhals result and concatenate with `pl_items` using `pl.concat`. No warning is emitted because polars (unlike PySpark) has no SparkSession barrier and can merge native `pl.DataFrame` and collected polars frames cleanly.

Output: One ~5-line code change in the polars branch of `_concat_failure_cases`, plus one new regression test file proving the fix.
</objective>
