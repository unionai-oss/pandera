<objective>
Extract the inline PySpark error-setting logic in `DataFrameSchemaBackend.validate` (ARCH-04) to a dedicated instance method `_handle_pyspark_validation_result`. The `is_pyspark` boolean detection stays — the PySpark protocol (set `check_obj.pandera.errors`, return original frame) is genuinely different from the standard narwhals contract (raise `SchemaErrors`), as explicitly noted in feedback_backend_specific_code.md. The refactor replaces two inline blocks (error path and success path) with single method calls.

Purpose: Eliminate the fourth and final `is_pyspark` dispatch violation by abstracting the inline protocol-difference logic to a named method with a docstring that explains WHY PySpark is different. The `is_pyspark` check itself remains because the protocol difference cannot be eliminated — only the inline duplication can.

Output: New `_handle_pyspark_validation_result` method on `DataFrameSchemaBackend`; `validate()` updated to call it from the error and success paths.
