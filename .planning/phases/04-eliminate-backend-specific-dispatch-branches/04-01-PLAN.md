<objective>
Eliminate the four PySpark-specific dispatch branches introduced during Phase
02-04 as pragmatic bug-fixes. Replace each with a properly-abstracted
alternative that does not sniff on frame types or module strings.

Four dispatch sites:
1. SC1: `base.py:run_check` — PySpark branch avoids `_materialize()`. Fix:
   extend `_materialize()` in utils.py to handle PySpark via `.first()`, then
   delete the dead `run_check` branch.
2. SC2: `base.py:_concat_failure_cases` — module-string sniffing detects
   PySpark. Fix: keep `_build_lazy_failure_case` items wrapped in narwhals so
   `_concat_failure_cases` can dispatch on `nw.Implementation` instead.
3. SC3: `components.py:check_dtype` — frame-implementation probe detects
   PySpark. Fix: replace with schema-driven `isinstance(schema.dtype,
   pyspark_engine.DataType)` check so dispatch is driven by what the user
   configured, not what backend is present.
4. SC4: `container.py:validate()` — inline `is_pyspark` error-setting blocks.
   Fix: extract to a `_handle_pyspark_validation_result()` method for
   readability, keep `is_pyspark` flag in `validate()`.
</objective>

  <name>Task 1 (SC1): Fix _materialize() for PySpark and remove dead run_check branch</name>
  <files>
    pandera/api/narwhals/utils.py
    pandera/backends/narwhals/base.py
  </files>
  <action>
    In `pandera/api/narwhals/utils.py`, extend `_materialize()` to handle
    PySpark nw.DataFrame inputs by using `.first()` (bounded collect) and
    wrapping the result via pyarrow into a narwhals DataFrame. The existing
    SQL-lazy branch calls `.execute()` which is not present on
    `pyspark.sql.DataFrame` — add a PySpark sub-branch BEFORE the generic
    `.execute()` path.

    Proposed implementation:
    ```python
    def _materialize(frame) -> nw.DataFrame:
        if isinstance(frame, nw.LazyFrame):
            return frame.collect()
        if isinstance(frame, nw.DataFrame) and _is_sql_lazy(frame):
            if frame.implementation in (
                nw.Implementation.PYSPARK,
                nw.Implementation.PYSPARK_CONNECT,
            ):
                import pyarrow as pa
                native = nw.to_native(frame)
                row = native.first()
                if row is None:
                    schema_names = frame.collect_schema().names()
                    return nw.from_native(
                        pa.table({k: [] for k in schema_names})
                    )
                return nw.from_native(
                    pa.table({k: [v] for k, v in row.asDict().items()})
                )
            return nw.from_native(nw.to_native(frame).execute())
        return frame
    ```

    Then in `pandera/backends/narwhals/base.py`, delete the PySpark-specific
    branch in `run_check` (lines around 143-154 in the current file) that
    bypasses `_materialize()`. After the fix, `_materialize()` handles PySpark
    uniformly and the branch is dead code. The unified path becomes:
    ```python
    passed = bool(_materialize(passed_lf)[CHECK_OUTPUT_KEY][0])
    ```
    for all backends including PySpark.

    Also update the `check_nullable` method in `components.py` (which already
    calls `_materialize(combined_lf.select(...))`) — no change needed there
    since `_materialize()` now handles PySpark automatically.
  </action>

  <name>Task 3 (SC3): Fix check_dtype to use schema-driven dispatch</name>
  <files>
    pandera/backends/narwhals/components.py
  </files>
  <action>
    In `pandera/backends/narwhals/components.py`, replace the frame-implementation
    probe in `check_dtype`:

    OLD:
    ```python
    is_pyspark = check_obj.implementation in (
        nw.Implementation.PYSPARK,
        nw.Implementation.PYSPARK_CONNECT,
    )
    native_pyspark_schema = (
        nw.to_native(check_obj).schema if is_pyspark else None
    )
    ```

    NEW:
    ```python
    from pandera.engines import pyspark_engine as _pyspark_engine
    uses_pyspark_dtype = isinstance(schema.dtype, _pyspark_engine.DataType)
    native_pyspark_schema = (
        nw.to_native(check_obj).schema if uses_pyspark_dtype else None
    )
    ```

    And replace all `if is_pyspark:` checks within the loop with
    `if uses_pyspark_dtype:`.

    This dispatches on what the user configured (a PySpark dtype in the
    schema) rather than what backend is present (the frame implementation).
    The semantics are equivalent in practice but the schema-driven approach
    is architecturally cleaner — the dtype mismatch check should be governed
    by the dtype being compared, not the frame type.

    Move the `from pandera.engines import pyspark_engine` import inside the
    method (already the pattern for `narwhals_engine`).
  </action>

  <name>Task 4 (SC4): Extract PySpark error-setting to _handle_pyspark_validation_result()</name>
  <files>

  <action>
    In `pandera/backends/narwhals/container.py`, extract the two inline
    `is_pyspark` error-setting blocks in `DataFrameSchemaBackend.validate()`
    to a new method `_handle_pyspark_validation_result()`.

    Add this method to `DataFrameSchemaBackend`:

    ```python
    def _handle_pyspark_validation_result(
        self,
        check_obj,
        error_handler,
        schema,
        has_errors: bool,
    ):
        """Record validation outcome on PySpark DataFrame via pandera accessor.

        PySpark uses a different validation contract from other backends:
        errors are set on check_obj.pandera.errors and the original frame is
        returned, rather than raising SchemaErrors. This matches the native
        PySpark backend contract and is required for the existing PySpark test
        suite to pass.
        """
        if has_errors:
            error_dicts = error_handler.summarize(schema_name=schema.name)
            check_obj.pandera.errors = error_dicts
        else:
            check_obj.pandera.errors = {}
        return check_obj
    ```

    Then update `validate()` to call this method at both the error and success
    exit paths:

    Error path (was):
    ```python
    elif is_pyspark:
        error_dicts = error_handler.summarize(schema_name=schema.name)
        check_obj.pandera.errors = error_dicts
        return check_obj
    ```
    Error path (becomes):
    ```python
    elif is_pyspark:
        return self._handle_pyspark_validation_result(
            check_obj, error_handler, schema, has_errors=True
        )
    ```

    Success path (was):
    ```python
    if is_pyspark:
        check_obj.pandera.errors = {}
        return check_obj
    ```
    Success path (becomes):
    ```python
    if is_pyspark:
        return self._handle_pyspark_validation_result(
            check_obj, error_handler, schema, has_errors=False
        )
    ```
  </action>
