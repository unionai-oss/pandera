<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| narwhals frame → native backend | `_materialize` calls `nw.to_native(frame).first()` then constructs a pyarrow table from PySpark `Row.asDict()`; the trust here is that PySpark `.first()` returns either a `Row` or `None` (Assumption A1 in RESEARCH.md). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `_materialize` PySpark sub-branch | accept | Internal refactor — no user-controlled input crosses this boundary. PySpark frames originate from validated user code paths; pyarrow Table construction from a single Row is bounded. |
| T-04-01-02 | Denial-of-service | `_materialize` PySpark sub-branch | accept | `.first()` triggers bounded execution (one row); not a DOS vector. Documented constraint: callers must never pass a non-aggregated full PySpark frame to `_materialize`. |
| T-04-01-SC | Supply-chain | pyarrow lazy import | accept | pyarrow is already a transitive narwhals dependency (RESEARCH.md "Environment Availability"); no new package install. |
</threat_model>
