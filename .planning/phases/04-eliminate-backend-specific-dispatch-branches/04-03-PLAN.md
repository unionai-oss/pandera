<objective>
Replace the frame-implementation probe in `check_dtype` (ARCH-03) with a schema-driven probe. Instead of `is_pyspark = check_obj.implementation in (nw.Implementation.PYSPARK, nw.Implementation.PYSPARK_CONNECT)`, use `uses_pyspark_dtype = isinstance(schema.dtype, pyspark_engine.DataType)`. The `str(pyspark_dtype) == str(schema.dtype)` comparison itself is correct and stays — only the detection mechanism changes. Also add an explanatory comment to the `tests/pyspark/test_pyspark_dtypes.py:61-64` empty-frame workaround so the rationale (verifySchema=False mismatch with multi-value rows against single-column schema) is preserved.

Purpose: Eliminate the third of four `is_pyspark` dispatch violations. Dispatching on "what the user configured in the schema" rather than "what backend is present" is semantically cleaner — a PySpark-typed schema needs PySpark-aware comparison regardless of how it is executed.

Output: Updated `check_dtype` with schema-driven dispatch; explanatory comment added to test fixture workaround.
</objective>
