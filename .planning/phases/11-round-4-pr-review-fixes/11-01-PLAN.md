<objective>
Align the PySpark Narwhals backend with the Polars and Ibis Narwhals backends: remove the `is_pyspark` accessor-protocol branch in `DataFrameSchemaBackend.validate()`, delete the `_handle_pyspark_validation_result` method, and remove the dead `nw.Implementation.PYSPARK` branch from `_materialize()`. Update the ARCH-04 architectural tests in `tests/narwhals/test_phase01_arch.py` to reflect the removal of `_handle_pyspark_validation_result`, and hoist inner imports that don't need to be inner (NIT-05).

Purpose: PR #2339 round-4 review (deepyaman, 2026-05-30) — the Narwhals backend is supposed to unify validation contracts. Having PySpark Narwhals attach errors to `df.pandera.errors` (matching the native PySpark contract) while Polars/Ibis Narwhals raise `SchemaErrors` defeats the unification. The Narwhals-backed PySpark backend should raise `SchemaErrors` like the other Narwhals backends. The `_materialize()` PySpark branch was previously documented as dead code (Phase 10, COMMENT-L2) — now we delete it.

**Intentional CORR-02 regression.** Phase 5's CORR-02 fix added `check_obj.pandera.add_schema(schema)` inside `_handle_pyspark_validation_result` so the PySpark Narwhals path matched the native PySpark accessor protocol (setting `df.pandera.schema`). Deleting `_handle_pyspark_validation_result` in this plan removes that `add_schema` call, so the PySpark Narwhals success/drop-invalid-rows path no longer sets the `df.pandera.schema` accessor. This is intentional and consistent with the unified contract: the Polars and Ibis Narwhals backends never call `add_schema` either — they just return the parsed frame with no accessor side effects. Plan 02 mitigates the test-side impact with backend-aware helpers (and `xfail(condition=CONFIG.use_narwhals_backend)` markers on any PySpark test that asserts `.pandera.schema` is set). If a future requirement re-mandates `.pandera.schema` for the narwhals path, the fix is a new `add_schema` call in the unified success path of `DataFrameSchemaBackend.validate()` (NOT a reintroduction of the PySpark-specific branch); that is explicitly out of scope for Phase 11.

Output: `pandera/backends/narwhals/container.py` with no `is_pyspark` branch, `pandera/api/narwhals/utils.py` with no PySpark branch in `_materialize`, updated `tests/narwhals/test_phase01_arch.py`.
</objective>
